JAX-RS template application

This is a template web application based on JavaEE 8 that runs on Apache Tomcat 9 with Java 14 or above.

The application is configured to expose RESTful resources through the JAX-RS reference implementation, Jersey (version 2, for JavaEE).

JAX-RS JSON serialization is handled by the Jackson JSON Provider (version 2, for JavaEE).

The application also shows:

  • how to perform the basic JAX-RS environment setup (class org.univaq.swa.framework.base.RESTApp)
  • how to implement a simple resource class (class org.univaq.swa.examples.resources.SimpleRes)
  • how to implement a simple login resource class to issue an access token (class org.univaq.swa.framework.security.AutenticazioneRes)
  • how to implement a simple WebApplicationException (class org.univaq.swa.framework.base.RESTWebApplicationException)
  • how to implement Jackson custom serializers and deserializers for Java classes (package org.univaq.swa.framework.jackson).
  • how to implement a custom exception mapper (class org.univaq.swa.framework.security.AppExceptionMapper)
  • how to implement a filter to handle CORS preflight requests (class org.univaq.swa.framework.security.CORSFilter)
  • how to declare a JAX-RS authentication annotation (class org.univaq.swa.framework.security.AuthLevel1)
  • how to implement a filter to handle JAX-RS authentication annotations (class org.univaq.swa.framework.security.AuthLevel1Filter)

The RESTful resources, grouped under the /rest prefix, and are the following:

GET rest/simple/14 404
